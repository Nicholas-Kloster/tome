Architecture Overview

Two separate worlds: the research pipeline (internal) and the distributed Go tool

RESEARCH PIPELINE — internal, not distributed
1. Nick runs oreilly-reader skill
→ Search: "Ollama deployment"
→ Read: ch3 (setup), ch5 (API), ch8 (auth)
→ Files saved: ollama-ch3.txt, etc.
2. Extraction prompt / Claude
→ default_port, api_paths[], auth_default
→ misconfig_patterns[], default_creds[]
→ shodan_dorks[], aimap_probe config
3. Output: platforms/ollama.json
→ committed to repo, reviewed, shipped
GO BINARY — what ships to the world
go install github.com/Nicholas-Kloster/tome@latest
tome profile ollama
→ full OSINT profile (JSON or table)
tome dorks ollama --format jaxen
→ paste-ready Shodan dork set
tome probe ollama --format aimap
→ aimap probe config JSON
tome list
→ all known AI/ML platforms
tome scan <ip> --probe ollama
→ live target check vs profile
